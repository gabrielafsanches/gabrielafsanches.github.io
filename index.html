HTML5 - entendendo a linguagem

Tag <head>

Uma configuração muito importante contida nessa tag é a codificação encoding ou charset — incluída na tag <meta>. Ela configura caracteres "especiais" (acento e cedilha).

Um dos valores mais comuns usados hoje em dia é o UTF-8, também conhecido como Unicode. Há outras possibilidades como o latin1, usada anntigamente.

O UTF-8 é a recomendação atual para encoding na Web por ser amplamente suportada em navegadores e editores de códigos, alpem de compatível com praticamente todos os idiomas do mundo.

A instrução do DOCTYPE!

O DOCTYPE não é uma tag, mas uma instrução especial que indica ao navegador qual versão do HTML deve ser utilizada para exibir na página. Quando não coloccamos essa instrução a página é exibida numa espécie de "modo de compatibilidade", onde algumas tags não funcionam totalmente corretas — principalmente as tags e estilizações mais atuais do HTML5.

Web Components

Consistem em "um conjunto de cinco tecnologias: Templates, Shadow DOM, Custom Elements, HTML Imports e Decorators", sendo que essa última ainda não possui uma especificação.

  • Templates: é um mecanismo para encapsular um conteúdo do lado do cliente que não é renderizado quando a página é carregada, mas que pode ser instanciadoposteriormente em tempo de execução usando o JavaScrpt.

    Pense no template como um fragmento de conteúdo, que é armazeenado para um possível uso futuro no documento. Enquanto o parser processa o conteúdo do elemento <template> ao carregar a página, isso apenas garante que o conteúdo é válido; porém, o conteúdo do elemento ainda não foi renderizado.

  • Shadow DOM: realiza o isolamento de elementos do DOM em trechos independentes evitando que estilos CSS e comportamentos JavaScript dos elementos alterem as características uns dos outros.
  • Custom Elements: permite que o desenvolvedor crie tags diferentes das convencionais. Seu intuito é encapsular as demais tags em uma personalizada, reutilizável, manutenível e manuseável.

    Essa tecnologia auxilia bastante na diminuição dos enormes aninhamentos de divs, além de possibilitar que os elementos personalizados herdem características e propriedades de tags nativas específicas — como a tag <button>, por exemplo.

  • HTML Imports: podemos importar documentos HTML inteiros para dentro de páginas de uma maneira semelhante à importação de um arquivo JavaScript ou CSS.

    O HTML Import cria uma nova requisição ao servidor para realizar o download do arquivo. Essa tecnologia auxilia no encapsulamento dos componentes em arquivos externos, possibilitando importá-los posteriormente.

  • Decorators: aplicam os templates com base em seletores CSS e JavaScript para criar mudanças visuais e comportamentais.

    O elemento "content" inserido dentro do template será substituído com o conteúdo do elemento de decoração. E através do atributo "select" é possível especificar a posição exata do elemento particular na marcação que será produzida.

    Com o CSS em conjunto com as Media Queries seria possível criar uma poderosa ferramenta de Responsive Web Design utilizando decoradores.

É importante ressaltar que os únicos navegadores que implementam as quatro tecnologias são o Google Chrome e o Opera.
